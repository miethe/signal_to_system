The Agentic SDLC Framework is a structured approach to redesigning software delivery workflows for organizations that have adopted — or intend to adopt — AI agents as first-class participants in their engineering process. It is not a product, a platform, or a methodology with a certification. It is a set of design principles and concrete patterns organized around a central claim: that agents change the optimal structure of the delivery system, and that organizations need a coherent framework for reasoning about what to change and in what order.

The framework organizes around four layers of the delivery system.

**Specification infrastructure** covers how work is defined with sufficient precision for agent execution. This includes task file formats, acceptance criteria standards, scope boundary conventions, and the practices that produce artifacts an agent can act on without fabricating missing context.

**Context infrastructure** covers how organizational knowledge is made available to agents at runtime. The canonical artifacts are CLAUDE.md files, architecture decision records, convention documents, and domain glossaries — the structured knowledge base that gives an agent operational grounding before it reads a line of source code.

**Verification infrastructure** covers how agent output is evaluated. This includes test architecture that provides fast, precise feedback within the agent loop, linting and static analysis that enforce conventions mechanically, and review practices calibrated for human oversight of agent-generated code.

**Operating model** covers how the human-agent delivery system is organized and managed: task decomposition practices, scope discipline, workflow patterns for different classes of work, and the leadership practices that maintain system coherence as agent capabilities evolve.

The framework is developed through practice — from working delivery systems, not from theoretical analysis of what should work. Patterns are included because they have been observed to improve outcomes in real delivery contexts. Patterns that have been tried and failed are documented as anti-patterns with the failure mode described, so organizations can avoid repeating the same experiments.

The framework is intended as a living artifact: updated as agent capabilities change, as better patterns emerge, and as the organizational practices that support agentic delivery become better understood.
