`@miethe/ui` is a React component library built for rapid construction of technical interfaces — dashboards, data displays, developer tools, and documentation sites. The library is not intended for general consumption. It encodes specific design decisions and compositional patterns that recur across projects in my own work, and is maintained primarily for that purpose.

The component set is oriented toward information density rather than marketing aesthetics: data tables with sort and filter, timeline components, code blocks with syntax highlighting and copy functionality, callout and annotation components, and layout primitives designed for content-heavy interfaces. Styling is built on a consistent token system that supports both light and dark modes without per-component overrides.

Components are written to be composable rather than configurable — the preference is for small, composable primitives over large, prop-heavy components that try to handle every case internally. This keeps the library usable without deep familiarity with its API, which matters when the primary user is also the primary developer.

The library is used across several projects including Signal to System, and evolves as new patterns emerge from those uses. It is not published to npm. The repository is public for reference.
